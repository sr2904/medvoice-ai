How it works

Backend architecture

1. Audio intake

Browser upload or microphone recording reaches the FastAPI backend.

2. Audio normalization

Backend converts audio into 16k mono WAV to make transcription stable.

3. Speech-to-text

Faster-Whisper transcribes speech locally on your machine.

4. Healthcare correction

Medication and symptom corrections improve raw STT output. Optional LLM cleanup can be enabled.

5. Triage engine

Rules detect refill requests, medication side effects, and urgent symptom language.

6. Patient timeline

Every call is stored in SQLite so clinicians have context between appointments.

Where to add your own APIs

For a stronger final demo, you can later swap the STT layer or post-processing layer. In the backend, edit app/stt.py for another transcription provider or enable LLM cleanup in .env and app/openai_postprocess.py.
